

# Создание блога, контактов и админ-панели

## Текущая ситуация
Страницы `/blog`, `/contact` и `/admin` отсутствуют. Ссылки в навигации ведут на 404.

## Что будет создано

### 1. Страница контактов (`/contact`)
- Информация для связи с ссылкой на Telegram: @nigismo
- Кнопка-ссылка на `https://t.me/nigismo`
- Единый стиль с остальным сайтом (Header + Footer)

### 2. Страница блога (`/blog`)
- Список статей с заголовком, датой и кратким описанием
- Данные хранятся локально в файле `src/data/blog-posts.ts`
- Одна-две демо-статьи для примера
- Страница отдельной статьи (`/blog/:slug`) с рендерингом Markdown-контента

### 3. Админ-панель (`/admin`)
- Простая авторизация по паролю (без Supabase -- хранение в localStorage)
- Пароль задаётся как константа в коде (можно позже перенести в env)
- Возможности:
  - Просмотр списка статей блога
  - Добавление новой статьи (заголовок, описание, контент)
  - Редактирование и удаление статей
- Данные блога сохраняются в localStorage браузера

### 4. Обновление маршрутов
В `App.tsx` добавятся маршруты:
- `/blog` -- список статей
- `/blog/:slug` -- отдельная статья
- `/contact` -- контакты
- `/admin` -- админ-панель

### 5. Обновление Footer
- Добавить ссылку на Telegram @nigismo в секцию контактов

---

## Технические детали

### Новые файлы
- `src/pages/Blog.tsx` -- список статей
- `src/pages/BlogPost.tsx` -- страница статьи
- `src/pages/Contact.tsx` -- контакты с Telegram
- `src/pages/Admin.tsx` -- админ-панель с CRUD для статей
- `src/data/blog-posts.ts` -- начальные данные блога
- `src/hooks/useBlogPosts.ts` -- хук для работы с localStorage

### Изменяемые файлы
- `src/App.tsx` -- новые маршруты
- `src/components/Footer.tsx` -- ссылка на Telegram

### Хранение данных
Блог-посты хранятся в localStorage. При первой загрузке инициализируются из `blog-posts.ts`. Админка читает и пишет в localStorage.

### Авторизация в админке
Простой пароль (например, `edupro2024`), проверка через форму входа. Сессия сохраняется в sessionStorage до закрытия вкладки.
